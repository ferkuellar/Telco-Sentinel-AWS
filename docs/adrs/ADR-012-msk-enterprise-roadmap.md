# ADR-012 - MSK as the Enterprise Streaming Roadmap

## Status

Accepted

## Context

The MVP should stay simple, but the target enterprise state needs a documented streaming evolution path.

## Decision

Document Amazon MSK or MSK Serverless as the future-state event backbone for broader Kafka-aligned platform evolution.

## Alternatives Considered

- keep Kinesis indefinitely
- self-manage Kafka
- defer future-state design

## Consequences

### Positive

- clearer maturity roadmap
- aligns with broader event platform patterns
- supports multi-team consumer models

### Negative

- future migration planning required
